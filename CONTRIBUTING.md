# Contributing

## Convenciones

- Mantener el repo agnóstico por dominio.
- No hardcodear nombres de cliente, tenancy u OCIDs reales.
- Toda nueva automatización debe apoyarse en:
  - manifiesto del proyecto
  - contrato MCP
  - estructura `workspace/`

## Flujo sugerido

1. Crear branch de trabajo.
2. Ejecutar setup local.
3. Validar con los workflows locales/documentados.
4. Abrir PR con contexto del cambio y evidencia.
